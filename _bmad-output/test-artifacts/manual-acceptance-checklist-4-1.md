## AC #1: 原始事件流（FR38）

**目标：** 验证 Debug Panel 显示未经 UI 处理的 SDK 原始 JSON 事件流

- [ ] **1.1 打开 Debug Panel**
  - 步骤：点击 toolbar 中的 ladybug 图标按钮
  - 预期：右侧弹出 320pt 宽的 Debug Panel，有动画过渡

- [ ] **1.2 默认显示事件流 Tab**
  - 步骤：Debug Panel 打开后查看默认 Tab
  - 预期：默认选中「事件流」Tab，显示 segmented Picker（事件流 / Token / 工具日志）

- [ ] **1.3 事件流包含时间戳和类型标签**
  - 步骤：发送消息触发 Agent 执行，观察 Debug Panel 事件流
  - 预期：每条事件显示 HH:mm:ss.SSS 格式时间戳 + 带颜色的类型标签（toolUse=蓝, result=绿, assistant=紫, system=灰）

- [ ] **1.4 可折叠 JSON 详情**
  - 步骤：点击某条事件右侧的 chevron 按钮
  - 预期：展开显示 pretty-printed JSON（包含 id、type、content、metadata、timestamp），再次点击折叠

- [ ] **1.5 排除 partialMessage 事件**
  - 步骤：Agent 流式回复时观察事件流
  - 预期：事件流中不出现 partialMessage 类型的中间事件

- [ ] **1.6 空状态**
  - 步骤：新建会话后立即打开 Debug Panel（未发送消息）
  - 预期：显示空状态提示"无事件数据"

## AC #2: Token 统计（FR39）

**目标：** 验证 Debug Panel 实时显示 Token 消耗统计和估算费用

- [ ] **2.1 切换到 Token Tab**
  - 步骤：点击「Token」Tab
  - 预期：显示 2x2 汇总卡片网格（总 Token、费用、Input、Output）

- [ ] **2.2 单次调用统计**
  - 步骤：Agent 完成一次完整执行后查看 Token Tab
  - 预期：汇总卡片显示正确的 Token 数和费用（格式 $X.XXXX）

- [ ] **2.3 多次调用累加**
  - 步骤：在同一会话中多次发送消息，观察 Token 统计
  - 预期：总 Token 和费用是所有调用的累加值

- [ ] **2.4 逐次调用明细**
  - 步骤：查看「调用历史」区域
  - 预期：每次调用显示编号、input/output Token、费用、耗时

- [ ] **2.5 无调用时的空状态**
  - 步骤：新建会话未执行任务时查看 Token Tab
  - 预期：所有卡片值为 0，调用历史显示"无 LLM 调用记录"
