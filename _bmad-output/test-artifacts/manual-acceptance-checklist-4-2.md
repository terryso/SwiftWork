## AC #2: API Key 管理（NFR6）

**目标：** 验证 API Key 的查看状态、更新和 Keychain 持久化

- [ ] **2.1 未配置状态显示**
  - 步骤：首次启动（未配置 API Key）时打开设置
  - 预期：API Key 区域显示红色 xmark + "未配置"，无遮罩 Key 显示

- [ ] **2.2 已配置状态显示**
  - 步骤：已完成 onboarding（API Key 已保存到 Keychain）时打开设置
  - 预期：API Key 区域显示绿色 checkmark + "已配置" + 遮罩 Key（如 `sk-ant-a****XYZ`）

- [ ] **2.3 输入新 API Key**
  - 步骤：在 "更新 API Key" 输入框中输入 `sk-ant-test-key`
  - 预期：输入框接受输入，输入内容默认被 SecureField 遮罩

- [ ] **2.4 显示/隐藏 API Key 切换**
  - 步骤：点击眼睛图标按钮
  - 预期：切换 SecureField ↔ TextField，图标在 eye ↔ eye.slash 之间切换

- [ ] **2.5 保存 API Key**
  - 步骤：输入新 API Key → 点击「保存更改」
  - 预期：状态更新为"已配置"，显示遮罩 Key，输入框清空

- [ ] **2.6 拒绝空 Key**
  - 步骤：清空输入框 → 点击「保存更改」
  - 预期：保存按钮灰色不可点击（disabled）

- [ ] **2.7 保存后立即生效**
  - 步骤：更新 API Key → 关闭设置 → 发送消息
  - 预期：新 Key 在下次 Agent 调用时生效
