## AC #1: 500+ 事件滚动帧率 ≥ 60fps

**目标：** 验证 LazyVStack 懒加载 + 虚拟化窗口在大量事件下保持流畅

- [ ] **1.1 基础滚动流畅性**
  - 步骤：发送消息让 Agent 执行多次工具调用，积累 50+ 事件。快速上下滚动 Timeline
  - 预期：滚动流畅，无明显卡顿或掉帧

- [ ] **1.2 大量事件滚动**
  - 步骤：让 Agent 执行复杂任务（如"递归搜索项目中的 TODO 注释"），积累 200+ 事件。快速滚动浏览
  - 预期：滚动流畅，事件卡片正常渲染

- [ ] **1.3 虚拟化占位验证（可选）**
  - 步骤：用 Xcode View Hierarchy Debugger 检查 LazyVStack 中的视图数量
  - 预期：非可视区域的事件视图未被实例化

## AC #2: 1000+ 事件分页加载，UI 不冻结

**目标：** 验证大会话加载时分页策略生效

- [ ] **2.1 大会话切换**
  - 步骤：创建一个含大量事件的历史会话（可通过脚本批量插入 SwiftData Event），然后切换到该会话
  - 预期：会话切换时 UI 不冻结，仅加载前 50 个事件

- [ ] **2.2 小会话全量加载**
  - 步骤：切换到少于 100 个事件的普通会话
  - 预期：所有事件正常加载和显示

## AC #3: 8 小时运行无内存泄漏

**目标：** 验证长时间使用后内存占用稳定

- [ ] **3.1 短期内存稳定性（替代验证）**
  - 步骤：与 Agent 持续对话，积累 100+ 事件。通过 Xcode Memory Graph 或 Activity Monitor 观察内存
  - 预期：内存占用随事件增长缓慢，不出现阶梯式跳跃

- [ ] **3.2 事件裁剪生效**
  - 步骤：持续对话积累 500+ 事件（可发送多条消息让 Agent 反复执行工具调用）。观察最早的事件是否从 Timeline 中消失
  - 预期：超过 500 条后，最早的事件被裁剪，内存占用趋于稳定

## 智能滚动行为

**目标：** 验证 follow-latest / manual-browse 双模式正确切换

- [ ] **4.1 默认自动跟随**
  - 步骤：发送消息给 Agent，观察 Agent 执行过程中的滚动行为
  - 预期：新事件到达时自动滚动到底部，始终显示最新内容

- [ ] **4.2 上滚切换到手动模式**
  - 步骤：Agent 正在执行时，手动向上滚动浏览历史事件
  - 预期：新事件到达时不再自动滚动，右下角出现"回到底部"按钮（圆形箭头图标）

- [ ] **4.3 "回到底部"按钮功能**
  - 步骤：点击右下角的"回到底部"按钮
  - 预期：Timeline 滚动到最新事件，按钮消失，恢复自动跟随模式

- [ ] **4.4 滚到底部自动恢复跟随**
  - 步骤：在手动模式下，手动滚动到接近底部
  - 预期：到达底部附近时自动切换回 follow-latest 模式，新事件再次自动滚动

- [ ] **4.5 按钮动画**
  - 步骤：触发上滚 → 出现按钮 → 点击回到底部 → 按钮消失
  - 预期：按钮出现和消失时有平滑的淡入淡出 + 滑动动画
